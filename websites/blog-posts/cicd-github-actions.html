CI/CD Pipeline dengan GitHub Actions

25 Februari 2026 7 min read

Mengapa CI/CD Penting?

Continuous Integration dan Continuous Deployment (CI/CD) adalah practice yang essential dalam modern software development. Dengan CI/CD, kita bisa automate testing dan deployment, mengurangi human error, dan mempercepat delivery cycle.

GitHub Actions Overview

GitHub Actions adalah platform CI/CD yang terintegrasi langsung dengan GitHub repository. Keuntungannya:

  • Free untuk public repositories
  • Easy configuration dengan YAML
  • Banyak pre-built actions di marketplace
  • Terintegrasi dengan GitHub ecosystem

Sample Workflow

Berikut contoh workflow untuk deploy static website ke S3:

name: Deploy to S3

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Configure AWS Credentials
        uses: aws-actions/configure-aws-credentials@v1
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: us-east-1
      
      - name: Deploy to S3
        run: |
          aws s3 sync ./websites s3://my-bucket --delete
          
      - name: Invalidate CloudFront
        run: |
          aws cloudfront create-invalidation --distribution-id ${{ secrets.DISTRIBUTION_ID }} --paths "/*"

Best Practices

  1. Use Secrets - Jangan hardcode credentials di workflow
  2. Cache Dependencies - Speed up build dengan caching
  3. Matrix Builds - Test di multiple environments
  4. Branch Protection - Require CI pass sebelum merge
