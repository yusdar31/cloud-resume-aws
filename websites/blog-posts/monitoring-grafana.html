Monitoring Infrastructure dengan Grafana

10 Maret 2026 8 min read

Introduction

Dalam era modern infrastructure, observability adalah kunci. Artikel ini akan membahas bagaimana membangun sistem monitoring yang robust menggunakan Grafana, Prometheus, dan AWS CloudWatch.

Architecture Overview

Kami menggunakan pendekatan multi-layer monitoring:

  • Infrastructure Layer: CPU, Memory, Disk I/O (Node Exporter)
  • Application Layer: Request counts, Latency, Error rates
  • Business Layer: Transaction volume, Active users

Setting Up Prometheus

Langkah pertama adalah mengkonfigurasi Prometheus untuk scrape metrics dari target nodes. Berikut adalah contoh konfigurasi `prometheus.yml`: 

global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'node_exporter'
    static_configs:
      - targets: ['localhost:9100']

Visualizing in Grafana

Setelah data terkumpul di Prometheus, kita dapat menghubungkannya ke Grafana sebagai Data Source. Dashboard yang efektif harus bisa menceritakan kondisi sistem dalam sekejap "glance".

Alerting Strategy

Jangan hanya monitor, tapi juga beri notifikasi. Gunakan Alertmanager untuk mengirim notifikasi ke Slack atau PagerDuty ketika metric melewati threshold kritis.
